#### 2.1.3 身份数据流

```
用户首次访问
  │
  ├─ 未登录 ──→ 仅浏览（不可投注）
  │
  └─ 微信登录成功
       │
       └─ fetch('/api/me') → 服务端解析 JWT
            ├─ vip: true, expiry > 今天 → VIP 用户
            └─ vip: false → 免费用户
```

**不再信任 localStorage 的 `biubiu_vip` 值。** 前端每次页面加载时调 `/api/me` 获取真实 VIP 状态并缓存到内存变量。

---

#### 2.1.4 实现关键点

1. **微信回调签发 JWT 时嵌入 vip 标记：**
   — `api/wx-callback.js` 签发 JWT 时，查 VIP 名单（环境变量 `VIP_OPENIDS`，逗号分隔的 openid 列表）
   — JWT payload: `{ openid, name, avatar, vip: true/false, vipExpiry: '2026-07-19' }`

2. **新增 `/api/me` 端点：**
   — 读取 `Authorization: Bearer <token>` 头
   — 解析 JWT，返回 `{ openid, name, avatar, vip, vipExpiry }`
   — 免费用户 `{ vip: false }`

3. **新建 `lib/auth.js` 鉴权中间件：**
   — `verifyToken(req)` → 解析 Bearer token → 返回 payload 或 null
   — 各 API 端点（matches/live/odds/events/report）调用此中间件
   — VIP 专属端点（live/events/report）检查 `payload.vip === true && payload.vipExpiry > now`

4. **前端 `loadUserIdentity()` 函数：**
   ```js
   async function loadUserIdentity() {
     const res = await fetch('/api/me', { headers: { Authorization: `Bearer ${token}` } });
     const data = await res.json();
     userVip = data.vip && new Date(data.vipExpiry) > new Date();
     userOpenid = data.openid;
   }
   ```

5. **MVP 阶段 VIP 名册存储：**
   — 环境变量 `VIP_OPENIDS=openid1,openid2,openid3`
   — admin.html 继续用 localStorage 管理，手动同步到环境变量
   — 后续升级：接入微信支付回调 → 自动写入

### 3.6 撤销机制 — 时间比例返还

**当前：** `userPoints += 5` — 固定返5分，无论何时撤。

**改为按撤销时间占撤销窗口的比例返还：**

```
返还比例 = 1 - (已过秒数 / 撤销窗口秒数)

已过 0秒  → 返还 100%
已过 15秒 → 返还 50%（30s窗口）/ 75%（60s窗口）
已过 30秒 → 返还 0%（30s窗口到期）
已过 60秒 → 返还 0%（60s窗口到期）
```

```js
// 当前：
userPoints += 5;  // ❌ 固定返5

// 改为：
const elapsed = (Date.now() - guess.time) / 1000;
const window = userVip ? 60 : 30;
const ratio = Math.max(0, 1 - elapsed / window);
const refund = Math.floor(guess.totalStake * ratio);
userPoints += refund;
// toast: "已撤销，返还 X 分（{ratio}%）"
```

**意义：** 越早撤返越多。误操作立刻撤全返。等半天才撤返很少。公平合理。

## 四、投注结算系统 — 自动结算

### 4.1 方案

**流程：** 比赛结束 → 获取真实比分 → 自动结算 → 积分到账。

```
比赛开赛时间 + 120分钟（约比赛结束）
       │
       ├─ fetch 比分数据（MVP：从预置数据/API 获取）
       │
       ├─ 遍历该场比赛所有用户的投注
       │    ├─ 按结算规则判断每注输赢
       │    ├─ 赢：userPoints += bet.payout
       │    └─ 输：0
       │
       ├─ 更新 userGuesses[matchId].final = true
       ├─ 更新 userGuesses[matchId].result = 'win'|'loss'
       │
       └─ 刷新排行榜 + 弹出结算通知
```

**MVP 版本：**
- 客户端在比赛结束后（开赛时间+120min）触发结算
- 比分来源：预置在 `data/worldcup-2026.json` 中逐场补充 `home_score` / `away_score`
- 管理员在 `admin.html` 录入比分作为兜底
- 后续升级：对接 SportsMonk / API-Football 实时比分

**判断规则：**

| 投注类型 | 赢条件 |
|----------|--------|
| 胜平负-主胜 | 主队进球 > 客队进球 |
| 胜平负-平局 | 主队进球 == 客队进球 |
| 胜平负-客胜 | 主队进球 < 客队进球 |
| 大小球-大 | 总进球 > 2.5 |
| 大小球-小 | 总进球 < 2.5 |
| 让球-主队 | (主队进球 + 让球数) > 客队进球 |
| 让球-客队 | (主队进球 + 让球数) < 客队进球 |
| 猜比分 | 完全匹配比分 |

### 4.2 结算代码骨架

```js
function settleMatch(matchId, homeScore, awayScore) {
  const guesses = getAllGuessesForMatch(matchId);
  guesses.forEach(g => {
    g.bets.forEach(bet => {
      const won = judgeBet(bet, homeScore, awayScore);
      if (won) {
        const userId = g.userId;
        addPoints(userId, bet.payout);
        bet.result = 'win';
      } else {
        bet.result = 'loss';
      }
    });
    g.final = true;
  });
  saveAllGuesses();
  renderLeaderboard();
  notifyUsers(matchId);
}
```
